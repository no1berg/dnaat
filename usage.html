Running DNAAT

To run the DNAAT tool, you have two primary options for sequence alignment: Global Alignment and Local Alignment. Both are executed from the command line.

1. Clone the Repository

First, clone the DNAAT repository from GitHub and navigate into the project directory:

git clone https://github.com/no1berg/DNAAT.git
cd dnaat

2. Choose an Alignment Algorithm

Depending on your needs, you can run either the global or local alignment script.

Global Alignment (Needleman-Wunsch)

Global alignment is useful for comparing two sequences of similar length. To run a global alignment, use the following command:

python global_alignment.py

This will run the script with example sequences ("GATTACA" and "GCATGCU") and produce an alignment score and a visual representation of the alignment grid.

Local Alignment (Smith-Waterman)

Local alignment is ideal for finding the best region of similarity between two sequences. To run a local alignment, use this command:

python aligner.py

This script will perform a local alignment on two longer example DNA sequences and display the optimal alignment path in a heatmap.

3. Performance Testing

The DNAAT toolkit includes C-accelerated functions for performance. You can test the performance difference between the Python and C implementations by running:

python test_performance.py

This script will benchmark the `fill_score_grid` function and report the speedup of the C version.

4. Dependencies

DNAAT is designed to be lightweight. The core functionality relies on the following Python libraries:

  • ctypes: For interfacing with the C alignment engines.
  • difflib: Used in the local alignment script.
  • numpy: For numerical operations and creating scoring grids.
  • pandas: For data structures to aid in visualization.
  • matplotlib: For plotting the alignment grids and paths.

Ensure you have these libraries installed in your Python environment. You can typically install them using pip:

pip install numpy pandas matplotlib
